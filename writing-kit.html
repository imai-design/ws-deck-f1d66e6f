# やってほしいこと
次の5つの観点で、それぞれ「良い点」と「直したほうがよい点」を挙げてください。
01 読者の検索意図：読者が知りたいことに答えられているか
02 構成：話の順番は分かりやすいか、抜けている話題はないか
03 情報の鮮度：古い情報や、今は違うかもしれない記述はないか
04 重複・冗長：同じことのくり返しや、長すぎる箇所はないか
05 E-E-A-T（＝書き手の経験・専門性・信頼性が伝わるか）：根拠や出典が必要な箇所はどこか

# 出力の形

F 1週間 実測シート（従来 vs AI後） 記録

AIを使うと「本当に速くなったか」「品質は落ちていないか」を、感覚ではなく数字で見るための記録です。速さと質を必ずセットで見ます。この表をCSVで保存

この記録の使い方（大事）。　これは 「AI（道具）と記事」の評価です。利用者一人ひとりの順位づけや、人員配置・続ける続けないの判断には使いません。速くなった時間は、利用者がもっと多くの記事に挑戦して工賃を増やすこと、指導員が無理なく見られるようにすることに使います。人を減らすためではありません。

始める前に1回だけ決めること

  1. 従来見込（分）の出し方を決める → AIを使わず作っていた頃の、同じくらいの難しさの記事の所要時間（直近の実績の平均か、数本だけ実測）。「たぶん30分」と記憶で埋めない。[要確認：従来見込の基準]
  2. 合格ラインを決める → 既定の案は「品質4項目がどれも3点以上」。下回ったら、速くできても「要再」。先に決めて、あとから動かさない。[要確認：合格ライン（既定=各項目3以上）]
  3. 従来記事のベースラインを数本だけ取る → AIなしの記事を同じ品質スコアで採点しておく（CSVの「ベースライン」行）。これが無いと「質が従来比で落ちていないか」を比べられません。

1週間後の見方（この順番で見る）

  1. 増えたか（いちばん大事） → 1人が1週間に仕上げられた本数が増えたか。指導員が直しに使った時間（手戻り）が減ったか。これが「増幅」できているかの本命です。
  2. 質は保てたか → 品質(/20)が、従来記事のベースラインと比べて下がっていないか。合格ライン以上だった本数（例：5本中4本）を見る。1本でも各項目3未満が出たら、その記事は「要再」のまま。
  3. 速くなったか → 5本の時短率を小さい順に並べ、真ん中の値（中央値）を主役にする（1〜2本の極端な記事に振られないため）。下書きだけでなく「AI計（仕上げ込み）」で本当に速くなったかを見る。
  4. つまずきTOP3を次の改善へ → 「つまずき・気づき」欄から多かったもの3つを、翌週のプロンプト1行追加・チェックリスト1項目更新に回す。
  5. 公開GOは人 → WordPress入稿・公開の最後のGOは必ず人。確認した人の名前を「公開GO確認者」欄に残してから公開する。

最終チェックリスト（GO前に必ず・上から順に）

1. 事実・数字・固有名詞の裏取り

  • ☐ 数字（割合・件数・金額・日付）を、一次情報（公式サイト・公的資料など [要確認：確認先リスト]）で確認した
  • ☐ 社名・人名・商品名・地名などの固有名詞を、正しい表記で確認した
  • ☐ [要確認] が残っている箇所は、確認できたら事実に直し、確認できないものは記事に書かないか担当に確認を回した
  • ☐ AIが作った（もっともらしいが裏が取れない）数字・固有名詞が紛れ込んでいないか確認した

※ 事実の確認は、利用者が一次確認を試み、確認できない物は [要確認] のまま残す。最終確認と責任は指導員が持つ。

2. 一次情報・独自体験が残っているか（AIに書けない差別化）

  • ☐ クライアントの実体験・現場の声・独自の見解など、AIには書けない「その記事ならではの中身」が入っている
  • ☐ どこにでもある一般論だけで埋まっていない（薄ければ加筆を依頼）

3. クライアントのトンマナ・NGワード

  • ☐ 指定のトーン（です・ます／硬さ・柔らかさ）に合っている
  • ☐ NGワード一覧 [要確認：NGワード] の言葉を使っていない
  • ☐ 呼び方・表記ゆれ（同じものを違う言葉で呼んでいないか）を統一した

4. 在宅医療メディア特有の注意（医療・健康の表現）

  • ☐ 病気が「治る」「効く」など、効果を断定する表現になっていない
  • ☐ 「絶対」「必ず」「最高」など、誇大に受け取られる表現を避けた
  • ☐ 医療・健康・薬に関わる表記（薬機法など。該当し得る法令の一例。[要確認／専門家確認]）で迷う箇所は、確認先 [要確認：医療表現の確認先] に確認した
  • ☐ 受診や治療をうながす場合、自己判断を勧める書き方になっていない（必要に応じて専門家への相談を促す）

※ 法令名・基準の数値は、この場で断定せず [要確認／専門家確認] のまま残し、確認できてから直す。

5. 個人情報の扱い

  • ☐ 実在する個人を特定できる情報（氏名・住所・連絡先・写真など）が、許可なく入っていない
  • ☐ 体験談・事例で個人が特定されないよう配慮されている（必要なら仮名・ぼかし）

6. 重複・コピペ・著作権

  • ☐ 他サイトや書籍の文章をそのまま写した箇所がない
  • ☐ 引用がある場合、引用とわかる形になっていて、出典（公式サイト・書籍名など [要確認：出典]）を明記した
  • ☐ 記事に画像・図表を使う場合、その権利・出典 [要確認] を確認した（生成画像・素材のライセンス含む）
  • ☐ クライアントの既存記事と内容が大きくかぶっていない（リライト元 [要確認：元URL] と見比べた）

7. メタ情報

  • ☐ タイトルが内容と合っていて、クライアントの指定（文字数など [要確認]）に沿っている
  • ☐ 説明文（ディスクリプション）が用意され、内容と合っている
  • ☐ 内部リンク（関連記事へのリンク）の有無・貼り先の指定 [要確認] を確認した

8. 最終GO（入稿・公開の承認）

  • ☐ 1〜7をすべて確認した
  • ☐ 入稿・公開・送信の最後のボタンは、指導員のGO後、立ち会いのもとで押す
  • ☐ 確認した指導員の名前を記録に残した（実測シートの「公開GO確認者」欄）[要確認：担当指導員名]
